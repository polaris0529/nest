## 5. 비기능 요구사항 (Non-Functional Requirements)

### 5.1 아키텍처 구성
| 구분 | 기술 스택 | 설명 |
|------|-----------|------|
| 서비스 레이어(백엔드) | Node.js + NestJS | API·비즈니스 로직, (추후 DB 연동) 서버 사이드 처리 |
| 화면(프론트엔드) | React (React DOM) | 사용자 UI·화면 구성, 클라이언트 렌더링 |
| DB·ORM | **현재 미사용** · 추후 PostgreSQL + Prisma | 관계형 DB 도입 시 Prisma로 스키마·마이그레이션·타입 안전 쿼리 적용 예정 |

- **현재 구조:** 프론트엔드(React) ↔ API ↔ 백엔드(NestJS) (DB 없이 기동 가능)
- **추후 구조:** 프론트엔드(React) ↔ API ↔ 백엔드(NestJS) ↔ Prisma ↔ PostgreSQL

- DB 없이도 앱 기동 및 대부분 기능 검증 가능(Prisma·API/역할 관련 기능은 DB 연결 시 활성화).
